\begin{adjustwidth}{1.5em}{}
\subsubsection{SPEED (ICLR'26 Poster)}

\begin{itemize}
  \item \textbf{Title:} SPEED: Scalable, Precise, and Efficient Concept Erasure for Diffusion Models
  \item \textbf{Authors:} Ouxiang Li, Yuan Wang, Xinting Hu, et al.
  \item \textbf{OpenReview:} \url{https://openreview.net/forum?id=aoEtzdRkGh}
  \item \textbf{Scores:} 8, 6, 6, 6
  \item Method: Null space, SVD, erased concept mapping to existing concepts.
\end{itemize}

\end{adjustwidth}

\begin{adjustwidth}{1.5em}{}
\subsubsection{HiRM (ICLR'26 Poster)}

\begin{itemize}
  \item \textbf{Title:} Localized Concept Erasure in Text-to-Image Diffusion Models via High-Level Representation Misdirection
  \item \textbf{Authors:} Uichan Lee, Jeonghyeon Kim, Sangheum Hwang
  \item \textbf{OpenReview:} \url{https://openreview.net/forum?id=JSjOlFLUsS}
\end{itemize}

\end{adjustwidth}

\begin{adjustwidth}{1.5em}{}
\subsubsection{EraseAnything (ICML'25)}

\begin{itemize}
  \item \textbf{Title:} EraseAnything: Enabling Concept Erasure in Rectified Flow Transformers
  \item \textbf{Authors:} Daiheng Gao et al.
  \item \textbf{arXiv:} \url{https://arxiv.org/abs/2412.20413}
  \item \textbf{Architecture:} FLUX (rectified flow transformer / MMDiT). Paper claims SD3 support but code is FLUX-only.
  \item Method: Bi-level optimization with LoRA on \texttt{add\_q\_proj}/\texttt{add\_k\_proj}. Lower level erases target concept (ESD loss + attention regularizer); upper level preserves unrelated concepts (LoRA reconstruction + reverse self-contrastive loss). 3.57\,MB per concept.
  \item \textbf{Note:} Only existing concept erasure work on flow-based T2I models. Could potentially be adapted to SD~3.5 (same MMDiT architecture).
\end{itemize}

\end{adjustwidth}


\begin{adjustwidth}{1.5em}{}
\subsubsection{Prototype-Guided Concept Erasure (CVPR'26)}

\begin{itemize}
  \item \textbf{Title:} Prototype-Guided Concept Erasure in Diffusion Models
  \item \textbf{arXiv:} \url{https://arxiv.org/abs/2603.08271}
  \item \textbf{Eval:} SD 1.4 (main), SDXL, SD 3.5 (compatibility test).
  \item Method: Training-free. k-means clustering on concept embeddings to form prototypes; steers generation away from prototypes using near-miss pairs. Supports multi-concept erasure and broad categories (I2P).
  \item \textbf{Note:} One of few training-free methods tested on SD~3.5. Reports Ring-A-Bell/P4D/UnDiff ASR on SD~3.5.
\end{itemize}

\end{adjustwidth}

\begin{adjustwidth}{1.5em}{}
\subsubsection{EraseAnything++ (arXiv'25)}

\begin{itemize}
  \item \textbf{Title:} EraseAnything++: Enabling Concept Erasure in Rectified Flow Transformers Leveraging Multi-Object Optimization
  \item \textbf{arXiv:} \url{https://arxiv.org/abs/2603.00978}
  \item \textbf{Eval:} Flux.1-dev (image), Open-Sora (video).
  \item Method: Extension of EraseAnything with multi-objective optimization (Pareto), reverse self-contrastive loss, and volumetric attention regularization for video. LoRA-based on MMDiT.
  \item \textbf{Note:} First concept erasure work on video diffusion models. Also tested entity/abstraction/relationship categories.
\end{itemize}

\end{adjustwidth}

\begin{adjustwidth}{1.5em}{}
\subsubsection{DVE -- Differential Vector Erasure (ICML'26 under review)}

\begin{itemize}
  \item \textbf{Title:} Differential Vector Erasure: Unified Training-Free Concept Erasure for Flow Matching Models
  \item \textbf{arXiv:} \url{https://arxiv.org/abs/2602.01089}
  \item \textbf{Eval:} Flux.1-dev.
  \item Method: Training-free. Computes differential velocity vector between anchor and erase concepts, projects out harmful direction from generation trajectory. No fine-tuning needed.
  \item \textbf{Note:} Best ASR on Flux across I2P/MMA/RaB. Training-free approach achieving SOTA on flow-matching models.
\end{itemize}

\end{adjustwidth}

\begin{adjustwidth}{1.5em}{}
\subsubsection{SURE -- Subcategorical Erasure (ICLR'26 under review)}

\begin{itemize}
  \item \textbf{Title:} Achieving Subcategorical Erasure in Text-to-Image Models
  \item \textbf{OpenReview:} \url{https://openreview.net/forum?id=1K7TSEe2x5}
  \item \textbf{Scores:} 2, 4, 4, 2
  \item \textbf{Eval:} SD 1.4.
  \item Method: Lipschitz regularization to ensure smooth model behavior around target category, enabling erasure of all subcategories using only the parent category label. Neutral eraser + preservation objective.
  \item \textbf{Note:} Addresses subcategory erasure problem (erase ``guns'' erases all gun types). Only SD~1.4.
\end{itemize}

\end{adjustwidth}

\subsection{Task 4: SPEED on SD 3.5 (Try 1)}

Goal: adapt SPEED's concept erasure from SD~1.4 (UNet) to SD~3.5-Medium (MMDiT).

\subsubsection{Architecture Mapping}

\begin{tabular}{@{} l l l @{}}
\toprule
& \textbf{SD 1.4} & \textbf{SD 3.5} \\
\midrule
Backbone & UNet & MMDiT (24 transformer blocks) \\
Edited layers & \texttt{attn2.to\_k/v} & \texttt{attn.add\_k/v\_proj} \\
Text encoder & CLIP (768-d) & CLIP-L + CLIP-G + T5 $\to$ 4096-d \\
Embedding dim & 768 & 4096 $\xrightarrow{\texttt{context\_embedder}}$ 1536 \\
\bottomrule
\end{tabular}

\subsubsection{Key Issue: Weight Delta Too Large}

With default SPEED hyperparams, the null space covers $>$98\% of the 1536-d embedding space (vs.\ 768-d in SD~1.4), causing destructive weight updates. Added \texttt{edit\_scale} parameter to control delta magnitude.

\begin{tabular}{@{} l c l @{}}
\toprule
\textbf{Config} & \textbf{$\Delta W / W$} & \textbf{Result} \\
\midrule
Default (scale=1.0, no retain) & 4.2$\times$ & Noise output \\
scale=1.0, retain=instance.csv & 2.5$\times$ & Still broken \\
scale=0.3, retain=instance.csv & 0.75$\times$ & Image OK but erasure messy \\
scale=0.1, retain=instance.csv & 0.25$\times$ & Image OK but Snoopy preserved \\
\bottomrule
\end{tabular}

\subsubsection{Best Result (scale=0.3 + retain set)}

Snoopy identity is disrupted but output becomes incoherent --- not a clean erasure.

\subsubsection{Takeaway}

SPEED's null-space projection does not transfer well to MMDiT. The joint attention architecture responds differently to selective weight editing than UNet cross-attention. Possible next steps:
\begin{itemize}
  \item Edit only later transformer blocks (preserve structural layers).
  \item Edit \texttt{context\_embedder} (global 4096$\to$1536 projection) instead of per-block layers.
  \item Adapt EraseAnything's LoRA-based approach (designed for MMDiT).
\end{itemize}
